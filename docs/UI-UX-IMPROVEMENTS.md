# Unimail 邮箱界面 · UI/UE 提升建议

作为产品级邮箱客户端的持续优化参考，以下为界面与体验方面的改进方向（部分已实现）。

---

## 一、已实现的提升

### 1. 多套主题
- **浅色 / 深色**：默认浅色与护眼深色。
- **海洋**：青蓝色系，清爽专业。
- **森林**：自然绿色，长时间阅读更舒适。
- **日落**：暖橙色调，柔和温暖。
- **北极 (Nord)**：冷色系，适合偏爱低饱和度界面的用户。
- **玫瑰**：柔和粉调，区分于常规灰白。
- **石板**：冷灰中性，偏专业/商务感。

主题在 **设置 → 外观与主题** 中可选，也可通过顶部栏的调色板图标快速切换，选择会持久化到本地。

### 2. 未读视觉强化
- 邮件列表中，未读邮件左侧增加 **小圆点** 标识，配合加粗发件人与主题，未读状态更易扫读。

## 二、建议的后续提升方向

### 1. 信息架构与层级
- **面包屑 / 上下文**：在列表上方或标题旁增加当前文件夹名称（如「收件箱」「已发送」），减少“不知道当前在哪”的困惑。
- **Header 层次**：主标题（邮箱/产品名）与次级操作（写邮件、主题切换）的视觉权重可再区分，避免与列表区抢眼。

### 2. 列表与阅读体验
- **骨架屏**：将「加载中…」改为邮件列表骨架屏，减少空白等待感。
- **空状态**：收件箱/文件夹为空时，可增加插画 + 简短引导（如「去写第一封邮件」），并链到写邮件。
- **列表密度**：在设置中提供「紧凑 / 标准 / 宽松」三种列表行高，满足不同屏幕与习惯。

### 3. 邮件详情与操作
- **正文排版**：对纯文本邮件可统一行高、字间距、段落间距，提升长文可读性；若支持 HTML 邮件，需做好安全过滤与基础样式隔离。
- **操作反馈**：星标、删除、移动等操作后，可增加轻量 Toast 或行内提示（如「已移至已删除」），避免用户不确定是否生效。

### 4. 写邮件与草稿
- **草稿提示**：从写邮件返回列表时，若有未发送草稿，可提示是否保存；或自动保存草稿并支持「继续编辑」。
- **富文本**：若后续支持富文本，工具栏需简洁、可折叠，并保证键盘与无障碍操作。

### 5. 无障碍与健壮性
- **焦点与键盘**：列表、详情、写邮件之间支持 Tab 与方向键导航；列表项、按钮具备清晰的 `focus-visible` 样式。
- **语义与 ARIA**：重要区域使用 `role`、`aria-label`、`aria-current` 等，便于读屏与自动化测试。

### 6. 性能与感知性能
- **虚拟列表**：收件箱邮件很多时，可对列表做虚拟滚动，减少 DOM 数量。
- **乐观更新**：星标、已读等操作可先更新 UI，再在后台同步，降低等待感。

## 三、主题使用说明

- 在 **设置** 页顶部「外观与主题」中点击任意主题卡片即可切换。
- 在 **邮箱首页** 顶部右侧点击调色板图标，可从下拉菜单快速切换主题。
- 主题会保存在浏览器本地（localStorage），下次打开自动应用。

如需新增主题，可在 `src/types/theme.ts` 中增加选项，并在 `src/app/globals.css` 中为对应 `[data-theme="xxx"]` 补充一套 CSS 变量（background、foreground、primary、sidebar 等）。
